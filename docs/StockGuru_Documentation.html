🏠What is StockGuru?

StockGuru is an educational trading simulation platform designed to help retail investors in India safely practice trading and understand market dynamics — without risking real money. It covers NSE/BSE equities, Crypto, and Forex markets.

Every trade decision comes with full transparency: the system explains why it entered, what gates passed or failed, what the market regime is, and what the expected risk/reward looks like — turning every simulation into a learning moment.

✅ Safety First: LIVE_TRADING_ENABLED = False is hardcoded and can never be changed. Real broker APIs are never touched unless the user builds a separate connector after 50+ paper trades with ≥65% win rate.

Step-by-step lessons to get up and running with StockGuru from zero.

📦 Tutorial 1: Install & Run StockGuru Locally ▼

Prerequisites

Python 3.10+, pip, and git installed on your machine.

Step 1 — Clone and install dependencies

cd C:\Users\Hp\projects\stockguru
pip install -r requirements.txt

Step 2 — Set your API keys

Create a .env file in the project root:

CLAUDE_API_KEY=sk-ant-your-key-here
PAPER_CAPITAL=500000
TELEGRAM_BOT_TOKEN=optional
TELEGRAM_CHAT_ID=optional

Step 3 — Start the server

python app.py

The Flask server starts at http://localhost:5050. Open your browser and navigate there to see the dashboard.

✅ You should see the live dashboard with agent status cards, a market scanner, and the paper portfolio panel.
🔄 Tutorial 2: Trigger Your First Analysis Cycle ▼

Using the Dashboard

Click the "Run Now" button on the dashboard. This calls POST /api/run-now and runs all 14 agents in sequence.

Using the API directly

curl -X POST http://localhost:5050/api/run-now

What happens

The AgentOrchestrator fires all registered agents in order: MarketScanner → Technical → News → Institutional → Options → ClaudeIntelligence → TradeSignal → RiskManager → PaperTrader.

Results appear in real-time on the dashboard via WebSocket. The paper portfolio panel updates automatically after any executed trades.

How to Add a New Symbol to the Watchlist

Call the watchlist API with a POST request:

curl -X POST http://localhost:5050/api/watchlist \
  -H "Content-Type: application/json" \
  -d '{"symbol": "TATAMOTORS.NS", "exchange": "NSE"}'

Verify the symbol was added:

curl http://localhost:5050/api/watchlist
ℹ️ Use Yahoo Finance suffix format: .NS for NSE, .BO for BSE, no suffix for Crypto (e.g., BTC-USD).

How to Manually Adjust Gate Thresholds

Default thresholds are in src/core/conviction_filter.py under ConvictionFilter.THRESHOLDS. To override without editing code, use the Phase 5 API to insert a DynamicThreshold row:

curl -X POST http://localhost:5050/api/self-healing/apply \
  -H "Content-Type: application/json" \
  -d '{
    "gate_name": "vix_check",
    "market_regime": "VOLATILE",
    "current_value": 30.0
  }'

This tightens the VIX gate to 30 specifically during VOLATILE regimes. The change is logged to dynamic_thresholds and loaded by Phase5ThresholdManager on the next trade cycle.

How to Review the Conviction Audit Log

Every trade decision (executed or rejected) is logged with gate-by-gate detail:

curl "http://localhost:5050/api/trade-decision-log?limit=20"

Each record shows which of the 8 gates passed/failed, the conviction level (HIGH/MEDIUM/LOW), and the full rejection reason if applicable. Use this to understand why a trade was blocked.

How to Configure Telegram Alerts

1. Create a Telegram bot via @BotFather and get your token.
2. Add to your .env:

TELEGRAM_BOT_TOKEN=your_bot_token

You will receive a morning brief every day at 08:00 IST and an evening debrief at 16:00 IST automatically via the scheduler.

💡Why an 8-Gate Filter?

Most retail traders lose money not because they have bad trade ideas, but because they execute low-conviction setups — trades where one indicator looks good but the full picture is mixed. The 8-Gate Conviction Filter forces holistic validation before capital is committed.

Each gate guards against a specific failure mode: Gate 1 prevents chasing overbought momentum, Gate 2 ensures enough real buying interest, Gate 3 requires multi-agent agreement to prevent single-model bias, Gate 8 stops trading during market panic. Only trades where the macro + micro + timing all align are executed.

ℹ️ Research shows multi-factor confirmation improves trade win rates by 20-30% compared to single-indicator strategies. StockGuru's filter has raised simulated win rate from a baseline of ~45% to 65%+ on high-conviction setups.
💡Why Self-Healing? (Phase 5 Rationale)

Markets change regimes. A VIX threshold that works in a calm bull market is too tight in a volatile one. Static thresholds decay in effectiveness over time. Phase 5 solves this by continuously measuring how each gate actually performs on historical trades and suggesting tighter or looser thresholds based on evidence.

Critically, Phase 5 uses pure Python statistics — no black-box ML models. Every recommendation is explainable: "Gate 7 (Sentiment) is rejecting 40% of trades in TRENDING regimes, but those rejected trades had a 70% win rate. Recommendation: relax threshold from 0.3 to 0.15 in TRENDING markets."

💡Market Regimes Explained
RegimeCharacteristicsGate AdjustmentsRisk Profile
TRENDINGVIX < 20, trend strength > 0.6, consistent directionRelaxed volume gate, tighter R:R to capture bigger moves2% position size, 1.5x ATR stop, 2.5:1 target
RANGINGVIX 18–25, mean-reverting, no clear trendStandard thresholds, stricter consensus requirement1.5% position size, 1.2x ATR stop, 1.8:1 target

💡The Educational Philosophy

Every feature in StockGuru is designed with a learning outcome in mind. When a trade is rejected, the system doesn't just say "rejected" — it logs which gates failed and why, allowing users to study the logic. When Phase 5 recommends a threshold change, it explains the statistical reasoning behind it.

The goal is not to generate winning trades for users — it's to help users understand what quality trade setups look like, how risk management works, and how market conditions affect strategy performance.
